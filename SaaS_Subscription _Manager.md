## **SaaS Subscription Manager – Requirements**

**Overview:**  
 A full-stack web application to manage user subscriptions for SaaS products. Users can subscribe, upgrade/downgrade plans, and view subscription status, while admins manage plans and monitor usage. The system follows clean code practices and proper frontend-backend separation.

### **Functional Requirements:**

- **User Accounts & Subscriptions:**  
 Users can create accounts and subscribe to available plans. Store user and subscription data in a database.  
- **Plan Management:**  
 Admins can create, edit, or delete subscription plans, including pricing and features.  
- **Upgrade/Downgrade:**  
 Users can switch between plans with automatic status updates and restrictions based on current plan.  
- **Mock Payment Logic:**  
 Implement simulated payment flow to process subscription changes without real transactions.  
- **Subscription Status Tracking:**  
 Track active, expired, or canceled subscriptions and notify users of changes.  
- **Admin Panel:**  
 Dashboard for managing plans, monitoring user subscriptions, and viewing statistics.  
- **Data & API:**  
 Relational database (MySQL/PostgreSQL) and RESTful APIs for subscription and plan management.  
- **Frontend & Backend Architecture:**  
 Frontend built with React.js; backend with Node.js/Express or PHP, modular and clean.  
- **Validation & Security:**  
 Validate user inputs, prevent invalid upgrades/downgrades, and secure endpoints.  
- **Deployment:**  
 Deployable online with proper environment and database configuration.
